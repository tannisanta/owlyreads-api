OwlyReads API

The OwlyReads API provides a solution to manage book and magazine inventories across franchise locations. It allows stores to keep information about titles, authors, editions, pricing, and availability up to date. Adding new books, updating details, or applying promotions is done centrally and reflected across all connected systems.

The API supports search, filtering, and retrieval of books by different criteria. This helps franchise stores manage stock efficiently and maintain consistent information across locations.

Use the OwlyReads API to:

  • Keep inventory data consistent across multiple locations
  • Update new arrivals, editions, and promotions automatically
  • Provide real-time inventory status to staff
  • Integrate with point-of-sale systems and e-commerce platforms
  • Ensure accurate and up-to-date book and magazine information across all stores

Authentication

To use the OwlyReads API, you first need an access token. This token identifies your account and allows you to make authorized requests.

The token is valid for 1 hour (3600 seconds).

Get access token

Request an access token using your OwlyReads profile credentials.
Include the token in the Authorization header as Bearer {token} for API calls.

Request Body schema: application/x-www-form-urlencoded
required
grantType
required
string

OAuth grant type. Use profileCredentials.

profileId
required
string

OwlyReads profile ID.

profileSecret
required
string

Secret key for your OwlyReads profile.

Changelog

The changelog lists all updates, enhancements, and removals in the API.

Changes to the OwlyReads Catalog API aim to minimize disruption. Some adjustments may be required depending on the change type.

Deprecated fields or features in the changelog serve as early notice. They will not be removed immediately but should be avoided in new integrations.

Non-breaking changes

  • Requests: Adding optional fields or changing required fields to optional.
  • Responses: Adding optional fields or changing optional fields to required.
  • Headers: Adding optional headers or adjusting header name case.
  • Field length: Expanding maximum length of existing fields.
  • Identifier format: Adjusting prefixes or identifier formatting.
  • Webhooks: Adding new event types (opt-in) or new payload fields.
  • Rate limiting: Announced at least one month in advance.

Breaking changes

  • Operation removal: Removing an API operation.
  • Requests: Removing/renaming fields, making optional fields required, removing oneOf.
  • Responses: Removing/renaming fields, changing status codes, removing oneOf.
  • Type changes: Changing field data type.
  • Headers: Adding/removing required headers.
  • Enums: Adding/removing enum values.
  • Errors: Changing error codes.
  • Validation: Adding stricter rules.
  • Auth: Changing authentication or authorization requirements.
